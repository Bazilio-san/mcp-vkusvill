# Authentication

This document covers the MCP server's own authentication (who may call this server). The upstream
VkusVill API is anonymous, so the proxy needs no credentials to fetch data.

## Default state

`webServer.auth.enabled` is `false` by default — the server accepts MCP calls without credentials.
Enable it for any networked deployment.

## MCP server auth methods

Configured under `webServer.auth` in `config/*.yaml`. Supported methods:

- **Permanent server tokens** — `permanentServerTokens: ['<token>', …]`. O(1) lookup, for
  service-to-service callers. Each token must be at least 20 characters.
- **Basic** — `Authorization: Basic base64(user:pass)`; set `webServer.auth.basic.username` /
  `password`.
- **JWT** — `Authorization: Bearer <token>`. Four operating modes via `webServer.auth.jwtToken.mode`:
  - `legacyAesCtr` (default) — HS256 issue with the `encryptKey` secret.
  - `embedded` — ES256/RS256 with a built-in IdP; publishes JWKS and `POST /oauth/token`.
  - `localKey` — verify against a public key on disk.
  - `remoteJwks` — verify against a remote IdP's JWKS endpoint (this server does not issue tokens).

## Minting JWT tokens

- `/admin` — the Admin Panel web UI (see [Admin Panel](./admin-panel.md)).
- `node scripts/generate-jwt.js -u <username> -ttl <duration>` — CLI.
- `/gen-jwt` skill inside Claude Code — see [SKILLS](./SKILLS.md).
- `POST /gen-jwt` — HTTP endpoint, only when `webServer.genJwtApiEnable: true` (default `false`).

Example, an admin-capable token valid 30 days:

```bash
node scripts/generate-jwt.js -u admin -ttl 30d -p "allow=gen-token"
```

## Passing credentials to the server

When auth is enabled, send the standard `Authorization` header with the MCP request:

```json
"headers": {
  "Authorization": "Bearer <jwt-token>"
}
```

Via `mcp-remote`, add `--header "Authorization:Bearer <jwt-token>"` — with **no space** after the
colon.

## Admin panel JWT requirement

When `adminPanel.authType` includes `jwtToken`, the admin panel (`/admin`) accepts a JWT **only if
its payload contains `allow: 'gen-token'`**. JWTs without this claim are rejected with `401`. This
blocks short-lived tokens issued for other purposes (for example, the token auto-generated by the
Agent Tester page) from being replayed to mint arbitrary long-lived tokens. `permanentServerTokens`
and `basic` admin auth are unaffected.

## Revocation

Revoked JWTs and usernames under `webServer.auth.revoked` are rejected everywhere (MCP endpoints,
admin panel, Agent Tester).
